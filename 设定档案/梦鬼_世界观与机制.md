# 《梦鬼》世界观与机制设定档案
> 版本：v0.2 | 最后更新：2026-06-19

---

## 一、核心概念

《梦鬼》是一个**寄生性存在**，以电子游戏为外壳，以人类的「愿望」为燃料，以「宿主」为现实锚点持续运作。

它不是单纯的游戏，也不是单纯的超自然实体——它是两者的叠加：
- 对外表现为一款1990年代末的PC游戏文件
- 对内是一套可以捕获意识、操控记忆、感知现实的寄生系统

---

## 二、游戏形式（开发目标）

- **类型**：Galgame（视觉小说）
- **平台**：PC
- **美术风格**：千禧年梦核（Dreamcore）——低饱和色调、过度曝光的室内光、模糊边缘、像素与手绘混用
- **Meta元素**：
  - 玩家以「2026年购入旧电脑」的现代人视角启动游戏
  - 游戏界面本身伪装成Win98风格桌面，存在系统报错、文件残缺、假死机等演出
  - 玩家与游戏内角色的边界在后期逐渐模糊
  - 某些选项会「失效」，某些文件会「消失」，某些存档会「被覆盖」

## 三、双时间轴结构

### 时间轴 A — 2000年（过去·游戏内）
五名高中生进入《梦鬼》游戏互杀，以「愿望」为入场券。

| 角色 | 愿望（表） | 备注 |
|------|-----------|------|
| 夏冰 | 考第一，逃离原生家庭 | 优绩主义者 |
| 小野 | 成为歌手，替家里还债 | 音乐梦想 |
| 白鸥 | 掩盖自己失手杀人的事实 | 带着罪进来的人 |
| 林智子 | 对外声称「无愿望」，真实愿望待补全 | 主动入局调查真相 |
| 第五人 | 身份待定 | 逐渐被边缘化/消失 |

**结局**：智子赢得游戏，触发宿主机制，记忆被清洗，以新身份被送回现实。

### 时间轴 B — 2026年（现在·Meta层）
玩家（你）买到一台二手Win98电脑，发现硬盘里残存的《梦鬼》游戏文件与碎片记录。

玩以为自己在「调查一个都市传说」，实际上是在重新经历自己忘记的过去。

## 四、宿主机制（定义版）

| 项目 | 内容 |
|------|------|
| **宿主是什么** | 梦鬼在现实世界的锚点，宿主存在=梦鬼持续运作 |
| **宿主如何产生** | 每轮游戏胜者自动成为新宿主 |
| **宿主的感知** | 无意识。宿主以普通人身份生活，不知道自己的身份 |
| **记忆清洗范围** | 陈述性记忆（episodic）被清洗，程序性记忆（procedural）残留——这是反转节点的机制基础 |
| **宿主寿命周期** | 约25-30年后，宿主即将自然死亡，梦鬼需要寻找新宿主，重新触发游戏 |
| **召回机制** | 游戏文件所在的载体（电脑）会通过「偶然」的方式流回宿主手中 |
| **上一任宿主** | 待补全（可与2000年之前的游戏循环关联） |

## 五、逻辑修复记录（v0.1→v0.2）

### 矛盾①：「林智子无愿望」vs「林智子忘了愿望」
**修复方案**：智子对外谎称无愿望，真实愿望是「终结游戏，救出第五人」。
- 第五人是她的挚友/家人，比她更早进入游戏，从未出来
- 她赢了游戏但两件事都没完成，愿望与记忆一同被清洗
- 2026年她感到的莫名缺失感，正是这个未竟愿望的残影

### 矛盾②：记忆清洗与反转节点冲突
**修复方案**：清洗的只是陈述性记忆，程序性记忆（肌肉记忆、直觉反应）残留。
- 「知道怎么破解机关却不知道为什么」→程序性记忆
- 「感觉听过这句话」→情绪记忆的残留（边缘系统记忆）
- 「镜中脸的重叠」→视觉识别的深层记忆

### 矛盾③：宿主机制与「召回」的矛盾
**修复方案**：宿主是被动的锚点，不需要主动运营游戏。26年后宿主衰老，梦鬼主动触发召回——电脑辗转流回不是偶然，是梦鬼操控了「偶然」。
